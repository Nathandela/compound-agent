## Phase 1: Critical Bug Fix (P0)

### `2f0` - Fix better-sqlite3 native bindings error

**Problem**: better-sqlite3 fails when compound-agent is installed as a dependency in another project.

**Approach**: Option D - Lazy-load SQLite with graceful degradation

```typescript
// Graceful degradation - JSONL still works without SQLite
let db: Database | null = null;
try {
  const Database = require('better-sqlite3');
  db = new Database(path);
} catch (e) {
  console.warn('SQLite unavailable, using JSONL-only mode');
}
```

**Why this approach**:
- No postinstall scripts (fragile across platforms)
- No optional dependencies (confusing)
- JSONL is source of truth anyway
- SQLite is just a cache/index

**Tests**:
1. SQLite available → normal operation
2. SQLite unavailable → JSONL-only mode works
3. Search falls back to keyword search when no FTS
4. Stats shows degraded mode warning
5. Integration: install in fresh project, verify works

**Files**: `src/storage/sqlite.ts`, `src/storage/index.ts`
